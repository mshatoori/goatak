# آگاهی وضعیتی در میدان نبرد

---

# چالش‌های میدان نبرد

- حجم عظیم داده‌های بلادرنگ
- مدیریت و توزیع اطلاعات
- سردرگمی و خطای تصمیم‌گیری
- تلفات خودی (Friendly Fire)
- محدودیت استفاده از سامانه‌های خارجی

---

# آگاهی وضعیتی چیست؟

- درک عناصر محیطی
- فهم معنای اطلاعات
- پیش‌بینی وضعیت آینده

**مدل سه سطحی:**

1. درک (Perception)
2. فهم (Comprehension)
3. پیش‌بینی (Projection)

---

# هدف تحقیق

طراحی و پیاده‌سازی سامانه بومی آگاهی وضعیتی تاکتیکی

- معماری غیرمتمرکز
- پشتیبانی از لینک‌های رادیویی ناپایدار
- تعامل‌پذیری با سامانه‌های نظامی

---

# نیازمندی‌های عملکردی

- ردیابی نیروهای خودی (BFT)
- ترسیم تاکتیکی
- پیام‌رسانی متنی
- اتصال به حسگرها
- هشداردهی و اعلام اضطرار
- مدیریت لایه‌ها

---

# نیازمندی‌های غیرعملکردی

- عملکرد غیرمتمرکز
- کارایی بالا
- قابلیت اطمینان
- امنیت
- قابلیت حمل

---

# معماری شبکه

**سطح ۱:** زیرشبکه‌های تاکتیکی (دسته/گروهان)

- ارتباط Peer-to-Peer / Broadcast

**سطح ۲:** گره‌های سرگروه

- تجمیع و مسیریابی اطلاعات

**سطح ۳:** مرکز فرماندهی

- دید کامل و آرشیو داده‌ها

---

# اجزای نرم‌افزاری

- **بک‌اند:** زبان Go
- **فرانت‌اند:** فریم‌ورک Vue.js

---

# فیدهای ارتباطی

- **ورودی:** فقط دریافت
- **خروجی:** فقط ارسال
- **دوطرفه:** ارسال و دریافت

---

# پروتکل‌های ارتباطی

- **UDP:** استاندارد TAK
  - Unicast / Broadcast / Multicast
- **RabbitMQ:** صف‌بندی پیام‌ها

---

# پروتکل CoT

**Cursor on Target:**

- استاندارد پیام‌رسانی XML
- تمرکز بر: چیست؟ کجاست؟ کی؟

**ساختار پیام:**

- UID، Type، Time، Point، Detail

---

# استاندارد MIL-STD-2525

- نمادهای نظامی ناتو
- نیروهای خودی، دشمن، غیرنظامی
- تسریع درک اطلاعات

---

# یکپارچه‌سازی حسگرها

**GPS:** اتصال به gpsd
**AIS:** ردیابی شناورها
**رادار:** نمایش اهداف

---

# سرویس‌های بک‌اند

- CoT Router
- Web Server
- Tracking Service
- Auth Service
- Resend Service

# پایگاه داده
